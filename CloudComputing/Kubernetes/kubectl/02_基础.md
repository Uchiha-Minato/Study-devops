# 基础 (起始、中间)

## create

`kubectl create --help`

从文件或标准输入(stdin)创建资源，JSON或YAML都可以。

**用法**

```bash
kubectl create -f FILENAME [options]
```

**可以创建的资源（部分）**

`role` `clusterrole` `rolebinding` `clusterrolebinding`

`configmap` `secret` `namespace` `serviceaccount` `token`
 
`deployment` `pod` `service` `job` `cronjob` `ingress` 

`poddisruptionbudget` `priorityclass` `quota` ...

### options 选项（常用）

- `--edit=false` 在创建前编辑（默认不编辑）
- `-f, --filename=[]` 用于创建资源的文件，可以是文件名、目录或url
- `-o, --output=''` 输出的格式。
    json, yaml, name, go-template, go-template-file, template, templatefile, jsonpath,
	jsonpath-as-json, jsonpath-file其中之一

**举例**
```bash
# 用pod.json中的数据创建一个pod
kubectl create -f ./pod.json
  
# 根据传入stdin的JSON创建一个pod
cat pod.json | kubectl create -f -
  
# 编辑registry.yaml中的数据。然后使用编辑过的数据创建资源
kubectl create -f registry.yaml --edit -o json
```

## expose

`kubectl expose --help`

将一个资源作为新的 Kubernetes 服务进行暴露。

可能的资源包括（不区分大小写）：

Pod （po）、service （svc）、replicationcontroller （rc）、deployment （deploy）、replicaset （rs）
