# 基础 (起始、中间)

*文中options不包含全部。如需查看更多可用选项，可以使用`kubectl COMMAND --help`。*

## create

`kubectl create --help`

从文件或标准输入(stdin)创建资源，JSON或YAML都可以。

**用法**

```bash
kubectl create -f 文件名 [options]
```

**可以创建的资源（部分）**

`role` `clusterrole` `rolebinding` `clusterrolebinding`

`configmap` `secret` `namespace` `serviceaccount` `token`
 
`deployment` `pod` `service` `job` `cronjob` `ingress` 

`poddisruptionbudget` `priorityclass` `quota` ...

### options 选项（常用）

- `--edit=false` 在创建前编辑（默认不编辑）
- `-f, --filename=[]` 用于创建资源的文件，可以是文件名、目录或url
- `-o, --output=''` 输出的格式。
    json, yaml, name, go-template, go-template-file, template, templatefile, jsonpath,
	jsonpath-as-json, jsonpath-file其中之一

**举例**
```bash
# 用pod.json中的数据创建一个pod
kubectl create -f ./pod.json
  
# 根据传入stdin的JSON创建一个pod
cat pod.json | kubectl create -f -
  
# 编辑registry.yaml中的数据。然后使用编辑过的数据创建资源
kubectl create -f registry.yaml --edit -o json
```

## expose

`kubectl expose --help`

将一个资源作为新的 Kubernetes 服务进行暴露。

可能的资源包括（不区分大小写）：

Pod （po）、service （svc）、replicationcontroller （rc）、deployment （deploy）、replicaset （rs）

* 注：`ReplicationController`为用于管理可水平扩展的工作负载的旧版 API。 被 Deployment 和 ReplicaSet API 取代。

**用法**
```bash
kubectl expose (-f 文件名 | 资源类型 资源名称) [--port=port] [--protocol=TCP|UDP|SCTP] [--target-port=number-or-name] [--name=name] [--external-ip=external-ip-of-service] [--type=type] [options]
```

### options

- `--cluster-ip=''` 要分配给服务的 ClusterIP。留空表示自动分配，或设置为 “None” 以创建无头服务。
- `--external-ip=''` 为服务所接受的其他外部 IP 地址（不由 Kubernetes 管理）。如果这个 IP 被路由到一个节点，除了其生成的服务 IP 外，还可以通过这个 IP 访问服务。
- `-f, --filename=[]` 标识要公开服务的资源的文件的文件名、目录或URL。
- `-k, --kustomize=''` 处理自定义目录。此选项不能与-f或-R一起使用。
- `-l, --labels=''` 应用于此调用创建的服务的标签。
- `--name=''` 新创建的对象（服务）的名称
- `-o, --output` 输出的格式。
    json, yaml, name, go-template, go-template-file, template, templatefile, jsonpath,
	jsonpath-as-json, jsonpath-file其中之一
- `--overrides=''` 针对所生成对象的内联 JSON 覆盖。如果这一对象是非空的，将用于覆盖所生成的对象。要求对象提供有效的 apiVersion 字段。
- `--override-type='merge'` 覆盖生成的对象的方法：json、merge或strategic。
- `--port=''` 服务要使用的端口。如果没有指定，则从被暴露的资源复制。
- `--protocol=''` 要创建的服务的网络协议。默认为 “TCP”。
- `-R, --recursive=false` 递归地处理-f，--filename中使用的目录。当想要管理在同一目录中组织的相关清单时非常有用。
- `--selector=''` 用于此服务的标签选择器。仅支持基于等值比较的选择器要求。如果为空（默认），则从replication controller或replica set中推断选择器。
- `--target-port=''` 此为端口的名称或端口号，服务应将流量定向到容器上的这一端口。此属性为可选。
- `--type=''` 服务类型：ClusterIP, NodePort, LoadBalancer, 或 ExternalName. 默认为ClusterIP。

**举例**
```bash
#为nginx deployment创建服务，服务端口为80，连接端口为8000的容器
kubectl expose deployment nginx --port=80 --target-port=8000

#为使用replica set复制的nginx创建服务，服务端口80，连接端口为8000的容器
kubectl expose rs nginx --port=80 --target-port=8000

#为replication controller创建名为rc-nginx的服务，replication controller的类型和名称由nginx-controller.yaml指定，它在端口80上提供服务并连接到端口8000上的容器
kubectl expose -f nginx-controller.yaml --port=80 --target-port=8000 --name=rc-nginx
```

## run

`kubectl run --help`

在pod中创建并运行一个特定的image。

**用法**
```bash
kubectl run 名称 --image=image [--env="key=value"] [--port=port] [--dry-run=server|client] [--overrides=inline-json] [--command] -- [命令] [参数...] [options]
```

## options

- *`--image=''`* 指定容器要运行的镜像。(几乎是必须)
- `--env=[]` 设置容器内的环境变量。
- `--port=''` 容器暴露的端口。
- `--command=false` 如果存在true和额外参数，则使用它们作为容器中的‘command’字段，而不是默认的‘args’字段。
- `--overrides=''` 针对所生成对象的内联 JSON 覆盖。如果这一对象是非空的，将用于覆盖所生成的对象。要求对象提供有效的 apiVersion 字段。
- `--restart='Always'` 此Pod的重启策略。合法值[Always, OnFailure, Never]。
- `--rm=false` 如果为true，将在容器退出时删除此pod，只在进入容器时有效。比如同时使用参数`--attach`或`-i/--stdin`。
- `--attach=false` 如果为true，等待Pod开始运行，然后像`kubectl attach`一样连接到Pod中的容器。 默认为false，除非设置了`-i/--stdin`，在这种情况下默认为true。使用`--restart=Never`返回容器进程的退出代码。
- `--force=false` 如果为true，则立即从API中删除资源并绕过优雅删除。请注意，立即删除某些资源可能导致不一致或数据丢失，需要确认。
- `-l, --labels=''` 使用`,`隔开Pod的label，会覆写之前的值。
- `-o, --output=''` 输出的格式。
    json, yaml, name, go-template, go-template-file, template, templatefile, jsonpath,
	jsonpath-as-json, jsonpath-file其中之一
- `-q, --quiet=false` 如果为true, 静默提示信息输出。
- `--annoations=[]` Pod的声明部分

**举例**
```bash
#启动一个nginx pod，暴露80端口
kubectl run nginx --image=nginx:v1 --port=80

#启动一个nginx pod, 而是用从JSON解析的部分值集重载spec
kubectl run nginx --image=nginx:v1 --overrides='{ "apiVersion": "v1", "spec": { ... } '

#启动一个pod，但是使用一个自定义的命令
kubectl run python-test --image=bge:v1 --command -- python -m main
```
